## Legacy Docs Policy

Many older docs are still kept for history/reference.
Treat these folders as legacy unless a new guide links to them:

- `docs/legacy/frontend/`
- `docs/legacy/scripts/`
- `docs/legacy/maintenance/`
- `docs/legacy/permissions/`
- `docs/legacy/features/`
- root-level summary docs like `*_SUMMARY.md`, `*_REVIEW.md`, `*_COMPLETE*.md`

They are useful for context, but not the default operating guide.
